# Reading ELLA Manual Review Guide

## 1. 문서 목적

이 문서는 자동 검수기(`validate_packs.py`)가 잡지 못하는 부분을
사람이 어떻게 확인할지 정하는 기준표다.

쉽게 말하면:

`코드는 형식을 보고, 사람은 품질을 본다.`

v1에서는 이 두 검수를 분리해서 운영한다.

## 2. 왜 이 문서가 필요한가

자동 검수기는 아래를 잘 잡는다.

- JSON 형식 오류
- 문항 수 오류
- 보기 수 오류
- 정답 번호 범위 오류
- 레벨별 skill 분포 오류
- 단어 수 / 문장 수 범위 경고

하지만 자동 검수기만으로는 아래를 정확히 판단하기 어렵다.

- 초1이 읽기에 너무 어려운 문장인지
- 보기 문장이 어색한지
- 정답이 정말 하나만 자연스럽게 맞는지
- 해설 한국어가 너무 어려운지
- 다른 세트와 너무 비슷한지

즉, 자동 검수 통과 = 배포 가능이 아니다.

## 3. 운영 원칙

v1 운영 원칙은 아래처럼 간다.

1. 스크립트로 초안을 만든다.
2. 자동 검수기를 돌린다.
3. 자동 검수 통과본만 수동 검수 표에 올린다.
4. 사람이 체크리스트로 읽어 본다.
5. `pass / revise / hold` 중 하나로 결정한다.

쉽게 말하면:

`auto pass -> manual review -> final decision`

## 4. 수동 검수 결과 상태

수동 검수에서는 아래 3개 상태만 쓴다.

### `pass`

- 학생용 배포 후보로 올릴 수 있음
- 큰 수정 없이 다음 단계로 넘어감

### `revise`

- 고치면 쓸 수 있음
- 문장, 보기, 해설, 중복 중 일부 수정이 필요함

### `hold`

- 지금은 배포 후보에서 뺌
- 너무 비슷하거나, 레벨이 심하게 안 맞거나, 품질이 낮을 때 사용

## 5. 자동 검수와 사람 검수의 경계

### 자동 검수 담당

- 형식 확인
- 숫자 규칙 확인
- 레벨별 skill 분포 확인
- word count / sentence count 경고

### 사람 검수 담당

- 초1이 실제로 읽을 수 있는지
- 질문이 자연스러운지
- 보기 4개가 서로 비교 가능한지
- 해설이 쉬운 한국어인지
- 다른 세트와 지나치게 비슷하지 않은지

## 6. 핵심 체크리스트

아래 5개를 반드시 본다.

### 6.1 초1 문장 난도

질문:

- 문장 하나에 정보가 너무 많이 들어가 있지 않은가
- 낯선 어휘가 연속으로 너무 많이 나오지 않는가
- 지문 흐름이 초1 기준으로 따라가기 가능한가
- level에 맞는 연결어만 자연스럽게 쓰였는가

#### 바로 수정해야 하는 신호

- 한 문장 안에 이유, 비교, 순서가 한꺼번에 들어가서 숨이 참
- 주어가 자꾸 바뀌어서 누가 무엇을 하는지 헷갈림
- 지문 이해에 꼭 필요하지 않은 어려운 단어가 많음
- 초1이 모를 만한 배경지식이 있어야 문제를 풀 수 있음

### 6.2 보기 어색함

질문:

- 정답이 정말 하나만 가장 자연스럽게 맞는가
- 오답이 너무 엉뚱하거나 너무 교묘하지 않은가
- 보기 4개 길이와 문장 톤이 너무 들쑥날쑥하지 않은가
- 보기 문법 형태가 서로 크게 깨지지 않는가

#### 바로 수정해야 하는 신호

- 정답이 너무 길거나 너무 자세해서 눈에 띔
- 오답 2개 이상이 거의 비슷하게 맞아 보임
- 보기 하나만 문법이 이상해서 정답처럼 보이거나 오답처럼 보임
- 보기 내용이 지문이 아니라 상식으로 풀리게 만듦

### 6.3 해설 쉬운 한국어 여부

질문:

- 한국어 해설이 초1 보호자나 운영자가 읽기에도 쉬운가
- `왜 정답인지`가 먼저 드러나는가
- GT / S / MGT에 맞는 설명 길이인가
- 어려운 문법 용어나 평가 용어를 남발하지 않는가

#### 레벨별 해설 기준

| 레벨 | 권장 길이 | 말투 기준 |
|---|---|---|
| `GT` | 1~2문장 | 아주 쉬운 한국어 |
| `S` | 2~3문장 | 쉬운 이유 설명 |
| `MGT` | 3~4문장 | 근거 + 오답 제거 이유 |

#### 바로 수정해야 하는 신호

- `함축`, `담화`, `추론적 단서` 같은 어려운 용어가 많음
- 정답 이유보다 오답 설명이 더 길어서 핵심이 흐림
- 한국어 문장이 너무 길어서 한 번에 안 읽힘
- GT인데 설명이 지나치게 딱딱하거나 길다

### 6.4 너무 비슷한 세트 제거

질문:

- 다른 pack과 주제만 같고 내용은 충분히 다른가
- 사건 순서, 질문 구조, 정답 근거가 또 복사된 느낌은 아닌가
- 이름이나 명사만 바꾸고 같은 문제를 반복한 것은 아닌가

#### 바로 보류(`hold`)해야 하는 신호

- 지문 구조가 거의 같고 명사만 바뀜
- 6문항 중 3문항 이상이 다른 세트와 사실상 같은 질문임
- 정답 근거 위치와 오답 방식이 거의 똑같음
- topic만 다르고 읽는 느낌이 같은 세트가 연속으로 나옴

#### 주의만 주면 되는 신호

- 같은 topic group 안에서 비슷한 소재가 다시 나옴
- 정답 패턴이 우연히 비슷함
- 가족, 학교, 동물처럼 원래 자주 나오는 주제가 겹침

중요:

- `같은 주제`와 `같은 세트`는 다르다
- 주제가 같아도 사건, 정보 구조, 질문 포인트가 다르면 유지 가능하다

### 6.5 validator가 못 잡는 항목

아래는 수동 검수에서 꼭 본다.

- 질문 문장이 어색한 번역투 영어인지
- 보기가 너무 길어서 읽기 피로가 큰지
- 지문 제목이 내용과 안 맞는지
- rationale이 실제 정답 근거와 맞는지
- inference 문항이 근거 없는 추측 문제로 변했는지

## 7. 레벨별 빠른 판단 기준

### GT

- 생활 주제 중심인가
- 직접 근거 찾기 위주인가
- 추론을 억지로 넣지 않았는가
- 번역이나 TTS 도움이 필요한 학생도 읽을 수 있는가

### S

- 두 단서를 연결하는 정도는 가능한가
- `because`, `but`, `then`이 나와도 무리 없는가
- inference가 1문항 수준으로 적절한가

### MGT

- 비교, 순서, 원인-결과가 자연스럽게 들어가는가
- inference가 2문항이어도 근거가 분명한가
- 오답이 그럴듯하지만 과도하게 함정형은 아닌가

## 8. 실제 검수 순서

권장 순서는 아래와 같다.

1. 자동 검수 결과를 먼저 본다.
2. 지문만 끝까지 읽는다.
3. 6문항을 답지 없이 먼저 읽는다.
4. 보기 4개가 서로 자연스럽게 비교되는지 본다.
5. 정답과 rationale을 본다.
6. 레벨 적합성과 중복 위험을 마지막에 판단한다.

이 순서를 쓰는 이유:

- 처음부터 정답을 보면 문항 품질 판단이 흐려지기 쉽기 때문이다

## 9. 검수 시트 사용 규칙

`scripts/create_manual_review_sheet.py`가 만드는 CSV에는 아래 개념이 들어간다.

- 자동 검수 상태
- 사람이 채워야 하는 상태 칸
- 문장 난도 / 보기 자연스러움 / 해설 쉬움 / 중복 위험 체크 칸
- 최종 판단 칸
- 메모 칸

권장 입력값:

- `sentence_level_ok`: `yes / no`
- `choice_naturalness_ok`: `yes / no`
- `rationale_korean_ok`: `yes / no`
- `duplicate_risk`: `low / medium / high`
- `review_status`: `pending / reviewing / done`
- `final_decision`: `pass / revise / hold`

## 10. 운영 예시

예시 흐름:

```text
1. python scripts/validate_packs.py packs
2. python scripts/create_manual_review_sheet.py packs
3. 운영자가 CSV를 열어 수동 검수
4. final_decision = pass 인 것만 학생용 후보로 이동
```

## 11. 결론

v1에서 중요한 것은
`자동 검수로 형식을 지키고, 수동 검수로 학생용 품질을 지키는 것`이다.

즉, 이 문서는 아래를 위한 기준표다.

- validator가 못 잡는 품질 문제를 찾기
- 초1 난도를 사람 눈으로 맞추기
- 비슷한 세트를 줄이기
- 배포 전 판단을 일관되게 하기
