# Rebuild v1 Build Plan

## 1. 문서 목적

이 문서는 새 폴더에서 실제 빌드를 시작할 때의 작업 순서를 정리한 문서다.

쉽게 말하면:
"무엇부터 만들고, 무엇은 나중으로 미룰지"를 정하는 문서다.

## 2. 기본 원칙

- 작은 단계로 나눈다
- 한 번에 모든 기능을 넣지 않는다
- 먼저 Reading MVP만 완성한다
- 프론트엔드 패키지 매니저는 `pnpm`으로 고정한다
- 백엔드는 FastAPI로 단순하게 시작한다
- 학생 기록은 SQLite로 저장한다
- 보조 학습 기능은 기본적으로 로컬 자산과 무상 기능부터 사용한다
- 코덱스 CLI는 프론트엔드 뼈대와 구조를 구현한다
- 본격적인 프론트엔드 디자인 구현은 안티그래비티에서 담당한다

## 3. 1차 마일스톤

1차 목표는 아래 하나다.

`Python으로 만든 문제와 번역 자산을 FastAPI가 읽고 채점하며, Next.js 웹앱이 문제를 풀고, 번역과 TTS를 선택적으로 사용하고, 오답과 팔로업 기록을 SQLite에 저장할 수 있게 만든다.`

## 4. 단계별 계획

## Step 1. 프로젝트 뼈대 만들기

### 작업

- 새 프로젝트 폴더 생성
- `packs/`, `scripts/`, `backend/`, `frontend/`, `docs/` 생성
- `translations/` 생성
- 문서 13종과 `TODO.md` 복사

### 프론트 명령

```bash
pnpm dlx create-next-app@latest frontend --ts --eslint --app --src-dir false --import-alias "@/*"
```

### 백엔드 시작 예

```bash
python -m venv .venv
.venv\Scripts\activate
pip install fastapi uvicorn
```

### 완료 기준

- `frontend/`가 생성되어 있다
- `backend/`가 생성되어 있다
- `pnpm install`이 정상 동작한다

## Step 2. Python 운영 도구 정리

### 작업

- `create_pack.py` 작성
- `validate_packs.py` 작성
- `level_profiles.py` 작성
- `create_translation_template.py` 작성
- 샘플 pack 3개 이상 준비

### 완료 기준

- 새 pack 템플릿 생성 가능
- 레벨별 템플릿 생성 가능
- 여러 pack 검증 가능
- 레벨 규칙 검증 가능
- 번역 템플릿 생성 가능

## Step 3. FastAPI 기본 뼈대 만들기

### 작업

- `backend/app/main.py` 작성
- 기본 health check API 작성
- CORS 및 기본 설정 정리

### 완료 기준

- FastAPI 서버가 기동된다
- 프론트에서 연결 테스트가 가능하다

## Step 4. SQLite 구조 먼저 고정

### 작업

- `attempts` 테이블 생성
- `attempt_answers` 테이블 생성
- `follow_up_items` 테이블 생성

### 완료 기준

- FastAPI 시작 시 DB 초기화가 가능하다
- 샘플 데이터 저장 테스트가 된다

## Step 5. 데이터 타입 먼저 고정

### 작업

- TypeScript에서 `Pack`, `SubmitRequest`, `SubmitResponse` 타입 작성
- FastAPI Pydantic 스키마 작성
- `04_DATA_CONTRACT.md` 기준으로 타입 확인

### 완료 기준

- 프론트 타입, 백엔드 스키마, 데이터 문서가 일치한다

## Step 6. 콘텐츠 보조 계층 만들기

### 작업

- `translations/{lang}/*.json` 구조 확정
- 지문 번역, 문항 번역, 보기 번역을 분리해서 다룰 수 있게 설계
- 로컬 번역 우선 로드 규칙 구현

### 완료 기준

- 번역 자산을 문제 원본과 분리해서 관리할 수 있다
- 부분 번역도 정상 처리된다

## Step 7. 문제 로드 기능 구현

### 작업

- FastAPI에서 `packs/*.json` 읽기
- FastAPI에서 `translations/*.json` 읽기
- 난이도별 선택 또는 랜덤 선택 구현
- 잘못된 형식이면 오류 처리

### 완료 기준

- 브라우저에서 문제 1세트를 가져올 수 있다
- 번역 가능 여부를 함께 전달할 수 있다

## Step 8. 문제 풀이 화면 구현

### 작업

- 시작 화면
- 문제 화면
- 6문항 선택 상태 관리
- 제출 전 미응답 검사
- 번역 토글 UI
- TTS 버튼 UI
- API 사용 토글 UI
- 디자인은 최소 뼈대 수준으로만 구현

### 완료 기준

- 사용자가 6문항을 모두 고를 수 있다
- 미응답 상태에서 제출이 막힌다
- 번역과 TTS를 켜고 끌 수 있다
- 안티그래비티가 본격 디자인을 입히기 좋은 구조로 정리되어 있다

## Step 9. 채점 및 결과 저장 구현

### 작업

- FastAPI에 제출 API 구현
- 결정적 채점 로직 구현
- 총점 계산
- 스킬별 점수 계산
- SQLite 저장

### 완료 기준

- 제출 후 총점과 스킬별 점수가 보인다
- 시도 기록이 SQLite에 저장된다

## Step 10. 보조 기능 연결 구현

### 작업

- 브라우저 TTS 연결
- 로컬 번역 표시 연결
- 외부 번역 API 호출 엔드포인트 연결
- 외부 오답 해석 API 호출 엔드포인트 연결
- 외부 TTS API 호출 엔드포인트 연결
- 모델 라우팅 규칙 연결
- `gemini-3.1-flash-lite-preview`를 기본 번역 / 짧은 힌트에 연결
- `gemini-2.5-flash`를 자세한 오답 해석에 연결
- API 토글 OFF 시 외부 호출 차단

### 완료 기준

- 브라우저 TTS가 기본 동작한다
- 로컬 번역이 있으면 바로 표시된다
- 결과 화면에서 오답 해석 요청이 가능하다
- 요청 목적에 따라 모델이 다르게 선택된다
- API 토글이 ON일 때만 외부 API를 쓸 수 있다

## Step 11. 오답 / 팔로업 기록 구현

### 작업

- 틀린 문항을 `follow_up_items`에 저장
- 최소 조회 API 구현

### 완료 기준

- 틀린 문제 목록을 나중에 다시 확인할 수 있다

## Step 12. 반복 검증

### 작업

- 샘플 pack 여러 개로 테스트
- 형식 오류 pack 테스트
- DB 저장 실패 상황 테스트
- API 토글 ON/OFF 테스트
- 번역 부분만 있는 경우 테스트
- 오답 해석 `short` / `deep` 분기 테스트
- 모델 라우팅 테스트

### 완료 기준

- 정상 흐름과 오류 흐름이 모두 확인된다

## 5. pnpm 규칙

프론트엔드와 관련된 모든 문서와 실행 예시는 `pnpm` 기준으로 적는다.

권장 명령:

```bash
pnpm install
pnpm dev
pnpm build
pnpm lint
```

필요하면 테스트 명령도 아래처럼 통일한다.

```bash
pnpm test
```

## 6. 2차로 미루는 항목

아래는 1차에서 만들지 않는다.

- 자동 번역
- Vocab 확장
- 최근 기록 대시보드
- 관리자 전용 화면
- 외부 API 보조 기능의 완전 자동 실행

## 7. 리스크와 대응

### 리스크 1. JSON과 DB 구조가 어긋날 수 있음

문제:
pack 구조와 저장 구조가 따로 놀 수 있다.

대응:
항상 `04_DATA_CONTRACT.md`를 기준으로 구현한다.

### 리스크 2. FastAPI와 프론트 연결이 꼬일 수 있음

문제:
타입과 응답 형식이 안 맞을 수 있다.

대응:
프론트 타입과 백엔드 스키마를 같이 맞춘다.

### 리스크 3. 기능 범위가 커질 수 있음

문제:
보조 기능을 너무 넓게 잡으면 일정이 무너진다.

대응:
1차는 Reading MVP + 선택형 번역/TTS + 오답 저장까지만 고정한다.

### 리스크 4. 외부 API가 무의식적으로 켜질 수 있음

문제:
사용자가 원하지 않는데 비용이 발생할 수 있다.

대응:
API 토글 기본값을 OFF로 두고, ON 상태를 화면에 분명히 표시한다.

## 8. 최종 완료 기준

아래가 모두 되면 1차 구현 완료로 본다.

- Python으로 새 문제 파일 생성 가능
- pack 검증 가능
- FastAPI 실행 가능
- `pnpm dev`로 프론트 실행 가능
- 웹에서 문제 로드 가능
- 6문항 풀이 가능
- 제출 가능
- 결과 점수 표시 가능
- 오답과 팔로업 데이터 SQLite 저장 가능
- 로컬 번역이 있으면 바로 표시 가능
- 지문/문항/보기 TTS 동작 가능
- API 토글 OFF일 때 외부 유료 API 호출 없음
