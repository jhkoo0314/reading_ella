# Rebuild v1 Level Guide

## 1. 문서 목적

이 문서는 초1 대상 Reading 서비스에서
`GT / S / MGT`를 어떤 기준으로 나눌지 정하는 문서다.

쉽게 말하면:
문제 난이도 운영 기준표다.

## 2. 가장 중요한 전제

이 문서의 레벨 구분은
**이 프로젝트에서 쓰는 내부 운영 기준**이다.

즉:

- 전국 모든 POLY 캠퍼스에 그대로 적용되는 공식 서열표가 아니다
- 공식 홈페이지가 공개한 큰 구조를 바탕으로
- 초1 서비스에 맞게 다시 단순화한 내부 기준이다

## 3. 공식 자료에서 확인되는 큰 구조

공식 ELE 페이지에서는 초등부가 아래처럼 소개된다.

- `GT`: Gifted & Talented
- `MAGNET`: `MAG`, `MGT`
- `Special`: `S1 ~ S3`

즉, 공식 자료 기준으로 초등부 핵심 축은
`GT / MAGNET / Special`로 보는 것이 가장 안전하다.

또한 공식 자료는 `MAGNET` 안에 `MGT`가 있고,
`Special` 안에 `S1 ~ S3`가 있다고 안내한다.

## 4. 캠퍼스 차이에 대한 전제

2025~2026 교육과정 안내 성격의 캠퍼스 공지에는
레벨과 캠퍼스별로 도입 여부가 다를 수 있다고 적혀 있다.

또 다른 캠퍼스 공지에서는 `GTi` 같은 이름도 실제로 보인다.

그래서 이 프로젝트는
"전국 공통의 딱 하나인 반 서열표"를 전제로 만들지 않는다.

## 5. v1에서 실제로 쓰는 레벨

v1에서는 아래 3개만 사용한다.

- `GT`
- `S`
- `MGT`

제외:

- `MAG`
- `GTi`
- `R`

제외 이유:

- `MAG`는 초1 Reading MVP 기준에서 범위가 너무 높고 넓다
- `GTi`, `R`은 캠퍼스별 운영 차이가 더 크다
- v1은 초1 핵심 트랙만 먼저 안정적으로 다루는 것이 더 중요하다

## 6. v1 내부 난이도 사다리

이 프로젝트는 초1 기준으로 아래 순서를 사용한다.

```text
GT -> S -> MGT
```

중요:

이 순서는 **이 프로젝트 내부 운영용 난이도 사다리**다.

즉:

- 공식 홈페이지가 전국 공통 서열로 직접 명시한 것은 아니다
- 서비스 운영과 문제 설계를 쉽게 만들기 위한 내부 기준이다

## 7. 레벨별 한 줄 정의

### GT

초1이 영어로 수업을 따라가되,
아직 번역과 TTS 도움이 자주 필요한 기본 상급 단계

### S

ECP 누적 학습 흐름을 이어 받아
읽기 안정감이 더 높은 초1 심화 단계

### MGT

읽기 속도와 정확도가 더 좋고,
문항 난도도 더 도전적으로 가져갈 수 있는 초1 상위 단계

## 8. 레벨별 문제 난이도표

| 레벨 | 학생 느낌 | 지문 길이 권장 | 문장 난도 | 보기 난도 | 추론 비중 |
|---|---|---:|---|---|---|
| `GT` | 읽을 수는 있지만 도움 기능이 자주 필요함 | 60~110 words | 짧고 직접적 | 정답 근거가 비교적 바로 보임 | 낮음 |
| `S` | 읽는 흐름이 안정적임 | 90~140 words | 연결어와 두 단서 결합 가능 | 비슷한 보기 1~2개 포함 | 중간 |
| `MGT` | 빠르고 정확하게 읽는 편 | 120~180 words | 이유, 비교, 지시어 해석 포함 가능 | 오답이 더 그럴듯함 | 높음 |

## 9. 레벨별 문항 구성 권장안

문항 수는 6개 고정이다.

### GT

- `main_idea`: 1
- `detail`: 3
- `vocab_in_context`: 2

해석:

- 초1 GT는 추론을 많이 넣기보다
- 직접 근거 찾기와 어휘 이해를 먼저 안정시키는 쪽이 맞다

### S

- `main_idea`: 1
- `detail`: 2
- `vocab_in_context`: 2
- `inference`: 1

해석:

- S는 직접 정보 찾기와 간단한 추론을 같이 다룰 수 있다

### MGT

- `main_idea`: 1
- `detail`: 2
- `vocab_in_context`: 1
- `inference`: 2

해석:

- MGT는 답을 바로 찾는 문제보다
- 두 단서를 연결하는 문제 비중을 더 늘릴 수 있다

## 10. 레벨별 지문 기준

### GT

- 한 가지 사건 또는 한 가지 정보 중심
- 친숙한 생활 주제 중심
- 현재시제, 짧은 문장 중심

### S

- 두 정보가 연결되는 구조 가능
- `and`, `but`, `because`, `then` 같은 연결어 자연스럽게 포함
- 짧은 이유 설명 가능

### MGT

- 비교, 순서, 원인-결과 구조 가능
- 지시어 해석 필요 가능
- 정보 밀도를 GT/S보다 높일 수 있음

## 11. 레벨별 보기 기준

### GT

- 보기 길이 짧게
- 오답은 너무 교묘하지 않게
- 정답 단서가 지문에 비교적 직접적으로 드러나게

### S

- 비슷한 보기 1~2개 포함 가능
- 두 단서를 같이 봐야 답이 보이는 문제 가능

### MGT

- 오답도 부분적으로 맞아 보이게 설계 가능
- 정확한 근거 비교가 필요하게 설계 가능

## 12. 레벨별 보조 기능 기준

### GT

- TTS 중요도 높음
- 번역 중요도 높음
- 오답 해설은 아주 쉬운 한국어 1~2문장

### S

- 번역은 선택 보조
- 오답 해설은 "왜 이 답인지"를 짧게 설명

### MGT

- 오답 해설에서 정답 근거와 오답 제거 이유를 함께 설명
- 필요하면 자세한 해설을 따로 요청할 수 있게 함

## 13. 레벨별 오답 해설 말투 기준

### GT

- `글에 바로 나와 있어요.`
- `이 문장을 보면 답을 찾을 수 있어요.`

### S

- `이 부분과 저 부분을 같이 보면 답이 보여요.`
- `한 문장만 보면 헷갈릴 수 있어요.`

### MGT

- `정답은 두 단서를 함께 봐야 합니다.`
- `오답은 일부만 맞고, 전체 뜻과는 다릅니다.`

## 14. 서비스 적용 규칙

v1 서비스는 아래처럼 적용한다.

- 시작 화면의 레벨 선택은 `GT / S / MGT`
- 기본 시작 레벨은 `GT`
- `MAG`는 UI에 노출하지 않음
- 내부 pack 메타데이터의 `level` 값은 `GT`, `S`, `MGT` 중 하나만 사용

## 15. 데이터 예시 권장안

pack id는 아래처럼 단순하게 가는 것을 권장한다.

```text
g1_{level}_{number}
```

예:

- `g1_gt_0001`
- `g1_s_0004`
- `g1_mgt_0007`

이유:

- 초1 대상이라는 점이 바로 보인다
- low/mid/high보다 실제 운영 의미가 더 분명하다

## 16. 이 문서의 결론

이 프로젝트는 초1 Reading MVP에서
아래 기준을 사용한다.

```text
사용 레벨: GT / S / MGT
제외 레벨: MAG / GTi / R
내부 난이도 순서: GT -> S -> MGT
```

다시 강조하면,
이건 **서비스 운영용 내부 기준**이다.
