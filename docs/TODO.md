## 5. 구현 경계 명시

아래 원칙은 반드시 지킨다.

- 코덱스 CLI는 프론트엔드에서도 먼저 뼈대와 구조만 구현한다
- 코덱스 CLI는 데이터 흐름, 상태, 접근성, 연결성을 우선 완성한다
- 코덱스 CLI는 최종 디자인 감도를 확정하지 않는다
- 안티그래비티는 코덱스가 만든 구조를 바탕으로 본격 디자인을 구현한다
- 안티그래비티는 기능 로직을 다시 짜는 것이 아니라 시각 완성도를 올리는 데 집중한다

개발자에게 전달할 한 줄 지시문:

`코덱스 CLI는 Python 스크립트, FastAPI, SQLite, Next.js 페이지 구조, 컴포넌트 뼈대, 상태 관리, API 연결, 기본 동작 UI까지만 구현하고, 본격적인 프론트엔드 디자인 구현과 최종 시각 완성도는 안티그래비티에서 담당한다.`

## 6. v1 콘텐츠 목표 수량

v1 문제 은행 목표는 아래로 고정한다.

- `GT`: 100세트
- `S`: 100세트
- `MGT`: 100세트

즉, 총 `300세트`를 기준 목표로 둔다.

중요:

- 한 번에 300세트를 모두 학생용으로 바로 배포하는 것이 목표는 아니다
- 먼저 대량 생성 가능한 구조를 만들고
- validator 통과본만 문제 은행에 넣는 것이 목표다
- 현재 `0001~0100` 자동 생성본은 실문항으로 다시 생성되었고 `strict validator`를 통과했다
- 다만 `scripts/create_pack.py` 자체는 여전히 작성용 템플릿 생성기다
